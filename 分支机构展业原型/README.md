# 分支机构展业提报系统 - 演示原型

## 📋 项目概述

这是一个基于HTML5 + JavaScript的**分支机构展业提报系统**完整演示原型，用于展业过程记录、企业信息维护、展业查询及汇总统计。

## 🎯 核心功能模块

### 1️⃣ **新增展业记录** 
位置：导航栏 → "新增展业记录"

**主要功能：**
- 填写展业基本信息（日期、分支、人员、联系方式）
- 选择展业企业（来自企业库，自动按分支过滤）
- 录入企业行业、发展阶段
- 详细描述展业项目/事项
- 记录推进状态
- 上传留痕材料（照片/附件，最多10个文件）
- 添加备注说明

**数据保存：**
- 自动生成记录ID
- 保存至localStorage（浏览器本地存储）
- 记录创建时间、最后更新时间

---

### 2️⃣ **企业信息维护** 
位置：导航栏 → "企业信息"

**核心特性（谁维护谁查看原则）：**
- 每个分支机构只能看到/维护自己分支的企业信息
- 当前演示用户所属分支：**北京分部**（可在代码中修改 `currentUserBranch` 变量）

**企业信息包含内容：**

| 分类 | 字段 |
|-----|------|
| **基本信息** | 企业名称、所属行业、发展阶段 |
| **关键信息** | 维护分支、维护人员、注册地址、成立时间 |
| **企业详情** | 法人代表、注册资本、企业资质、专利情况 |
| **业务信息** | 核心团队、行业地位、核心产品、业务需求、已开展业务 |
| **财务数据** | 2023-2025年 主营收入、净利润、总资产、净资产 |

**操作功能：**
- ➕ **新增企业** - 填写完整企业信息
- 📝 **编辑企业** - 修改已维护的企业信息
- 🗑️ **删除企业** - 删除企业记录
- 🔍 **查询过滤** - 按分支、企业名称、行业查询

---

### 3️⃣ **展业查询**
位置：导航栏 → "展业查询"

**查询条件：**
- 日期范围（开始日期 ~ 结束日期）
- 分支机构（可选多个）
- 负责人员（支持模糊搜索）
- 企业名称（支持模糊搜索）

**展示内容：**
- 展业日期、分支机构、负责人员
- 企业名称、事项摘要、推进状态
- 最近更新时间

**详细查看：**
- 点击"查看"按钮 → 查看完整记录信息 + 变更历史
- 变更历史以时间线形式展示，记录所有修改内容

**编辑功能：**
- 点击"编辑"按钮 → 修改展业记录
- 系统自动记录修改内容到变更日志

---

### 4️⃣ **汇总统计**
位置：导航栏 → "汇总统计"

**统计维度：**
- 按分支机构统计
- 按负责人员统计

**统计指标：**
| 指标 | 说明 |
|-----|------|
| 当日新增 | 当天新增的展业记录数 |
| 已完成 | 状态为"已完成"的项目数 |
| 推进中 | 状态为"推进中、已联系、待决策"的项目数 |
| 待决策 | 正在等待决策的项目数 |
| 完成率 | 已完成 / 总数 × 100% |

**统计卡片显示：**
- 当日新增展业记录数
- 当日已完成项目数
- 推进中项目数
- 待决策项目数

## 🔐 权限机制（谁维护谁查看原则）

### 企业信息权限：
```
维护权限：
  ├─ 只能维护自己分支的企业
  ├─ 只能编辑自己维护的企业
  └─ 只能删除自己维护的企业

查看权限：
  └─ 只能看到自己分支的企业列表
```

### 当前登录模拟：
系统模拟当前登录用户为 **北京分部** 的员工

如需切换为其他分支查看，修改代码第7行：
```javascript
let currentUserBranch = '北京分部'; // 改为其他分支名
```

## 💾 数据存储

### 存储位置：
- 浏览器 **localStorage**
- 数据自动保存，关闭浏览器后仍可恢复

### 存储的数据：
| 名称 | 键 | 说明 |
|-----|----|----|
| 展业记录 | `records` | JSON数组 |
| 企业信息 | `companies` | JSON数组 |

### 清空数据方法：
在浏览器控制台执行：
```javascript
localStorage.clear();
location.reload();
```

- Chrome 80+

## 📌 注意事项

1. **企业管理权限严格执行**：
   - 只能看到自己分支维护的企业
   - 在新增展业记录时自动显示当前分支的企业列表

2. **数据验证**：
   - 必填字段校验
   - 同分支内企业名称唯一性检查
   - 附件上传数量限制（≤10个）

3. **自动留痕**：
   - 每条展业记录修改都自动记录到变更历史
   - 记录操作人、操作时间、变更内容

4. **响应式设计**：
   - 支持桌面和平板查看
   - 移动端部分功能可能需要优化
